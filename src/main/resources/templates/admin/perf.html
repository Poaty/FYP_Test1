DB queries per render (manual verification)

The /feed and /for-you endpoints use JOIN FETCH queries that return all needed data in a fixed number of SQL statements (typically 2–3 per render), regardless of how many posts are in the database. This is by design — the alternative (lazy-loading per row) would N+1 with the feed size.

To verify for the report:

  1. Set spring.jpa.show-sql=true in application.properties temporarily.
  2. Restart, hit /feed once, count the SQL statements logged.
  3. Repeat with the DB at different sizes (10, 50, 100 posts) — count should remain constant.

We don't track this counter automatically because Hibernate's request-scoped statistics are awkward to capture cleanly — the manual count is more trustworthy and good enough for the report.
